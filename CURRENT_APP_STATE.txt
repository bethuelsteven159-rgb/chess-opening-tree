==================================================
2. AUTH + OFFLINE BEHAVIOR
==================================================

The app is still owner-only.

Allowed email:
- `bethuelsteven159@gmail.com`

Protected pages use the stored Supabase session first.

Practical effect:

- if you stay logged in, the installed app can keep working offline
- if you log out, offline access stops until you log in again online

Offline-capable modules right now:

- move editor
- training
- repair loop
- game analysis
- position vault
- Support Hub
- backup export

When Supabase is unavailable:

- local saves still succeed
- pending sync copies are preserved
- recovery snapshots are preserved
- the app refuses to wipe local data with empty remote state


==================================================
3. DATA COLLECTIONS
==================================================

The app now works with 13 logical collections:

1. `opening_nodes`
2. `repair_items`
3. `games`
4. `game_annotations`
5. `positions`
6. `mistakes`
7. `support_cards`
8. `goals`
9. `app_reminders`
10. `books`
11. `book_notes`
12. `tournament_notes`
13. `quick_ideas`

All 13 collections use the same safety pattern in `js/db.js`:

- current local copy
- non-empty snapshot copy
- pending sync copy
- guarded remote load
- guarded remote save

Important protection behavior:

- non-empty remote data is not blindly replaced by empty payloads
- if remote data is empty but local data exists, local data is preserved
- if pending local sync does not match remote state, pending local data is preserved
- connectivity failure keeps the latest local copy

12. SQL / REMOTE REQUIREMENT
==================================================

To sync the current build remotely, the user still needs to run:

- `supabase/schema.sql`

That script now creates or upgrades:

- `games`
- `game_annotations`
- `positions`
- `mistakes`
- `support_cards`
- `goals`
- `app_reminders`
- `books`
- `book_notes`
- `tournament_notes`
- `quick_ideas`

If the SQL is not run yet:

- the new pages still work locally
- sync for the new collections stays local-only
- the app warns that the remote tables are missing


==================================================
13. WHAT THIS BUILD REALLY CHANGED
==================================================

Before this sprint, the app could:

- store opening moves
- train opening moves
- store repair notes
- analyze games
- extract positions and mistakes

After this sprint, the app can now also:

- store goals and reminders
- pin mindset / anti-tilt / principle cards
- keep books and extraction notes
- hold tournament preparation and reflection
- capture and convert loose ideas
- surface the non-board work on the dashboard

This means the app is no longer only an opening and analysis workspace.

It is now a broader chess-brain workspace with a dedicated support layer around the board work.
